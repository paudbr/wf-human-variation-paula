# wf-human-variation-paula

> **Fork of [epi2me-labs/wf-human-variation](https://github.com/epi2me-labs/wf-human-variation)**
> Extended with a custom [deeptools](https://deeptools.readthedocs.io/) process in the modified base calling subworkflow (`--mod`) to visualise and compare methylation differences between haplotypes.

Human SNV, SV, CNV, STR genotyping and modified base calling — with haplotype-resolved methylation plots.

---

## What's different in this fork?

The original EPI2ME workflow already outputs haplotype-resolved bedMethyl and bigWig files when `--mod --phased` are used. This fork adds a downstream **deeptools step** inside `wf-human-mod.nf` that automatically:

1. Takes the per-haplotype bigWig files (`*.wf_mods.1-5mC.bw` and `*.wf_mods.2-5mC.bw`) generated by modkit.
2. Runs `bamCompare` / `bigwigCompare` (deeptools) to compute the log2 ratio or difference of 5mC signal between haplotype 1 and haplotype 2.
3. Produces a genome-wide **haplotype difference bigWig** (`{{ alias }}.wf_mods.haplo_diff.bw`) for direct visualisation in IGV or similar genome browsers.
4. Optionally generates a **summary heatmap/profile plot** with `plotHeatmap` / `plotProfile` over a set of regions of interest (e.g. a supplied `--bed` file), making it straightforward to compare allele-specific methylation patterns at a glance.

> **Requires:** `--mod --phased` to be enabled. deeptools must be available in the execution environment (it is included in the container used by this fork).

### New output files (mod subworkflow only)

| File | Description |
|------|-------------|
| `{{ alias }}.wf_mods.haplo_diff.bw` | bigWig of the 5mC signal difference (H1 − H2) across the genome |
| `{{ alias }}.wf_mods.haplo_heatmap.png` | deeptools heatmap of haplotype methylation differences over regions of interest |
| `{{ alias }}.wf_mods.haplo_profile.png` | deeptools profile plot of haplotype methylation differences over regions of interest |

## Introduction

This repository contains a [Nextflow](https://www.nextflow.io/) workflow for analysing variation in human genomic data. Specifically this workflow can perform the following:

* diploid variant calling
* structural variant calling
* analysis of modified base calls (+ haplotype methylation comparison — **new in this fork**)
* copy number variant calling
* short tandem repeat (STR) expansion genotyping

The workflow consolidates small variant calling (previously wf-human-snp), structural variant calling (wf-human-sv), and CNV calling (wf-cnv), all of which are now deprecated upstream, as well as STR expansion genotyping. All steps run simultaneously and results are output in the same way as they would have been had the pipelines been run separately.

[![wf-human-variation overview schematic.](docs/images/wf-human-variation.drawio.svg)](docs/images/wf-human-variation.drawio.svg)

The tools embedded in individual sub-workflows within wf-human-variation are specifically designed for use with whole-genome Oxford Nanopore Technologies sequencing data. While 20x average coverage is the absolute minimum requirement for the workflow to run, we recommend an average coverage above 30x to ensure optimal performance.

## Compute requirements

**Recommended:**
- CPUs = 32
- Memory = 128 GB

**Minimum:**
- CPUs = 16
- Memory = 32 GB

**Approximate run time:** Variable depending on coverage and the analyses requested. For instance, a 90X human sample run (`--snp --sv --mod --str --cnv --phased --sex XY`) takes less than 8 h with recommended resources.

**ARM processor support:** False

## Install and run

These are instructions to install and run the workflow on the command line. You can also access the original workflow via the [EPI2ME Desktop application](https://labs.epi2me.io/downloads/).

The workflow uses [Nextflow](https://www.nextflow.io/) to manage compute and software resources. The workflow can currently be run using either [Docker](https://docs.docker.com/get-started/) or [Singularity](https://docs.sylabs.io/guides/3.0/user-guide/index.html).

To run **this fork** directly:

```bash
nextflow run paudbr/wf-human-variation-paula --help
```

To run the upstream EPI2ME release:

```bash
nextflow run epi2me-labs/wf-human-variation --help
```

To update to the latest version:

```bash
nextflow pull paudbr/wf-human-variation-paula
```

### Demo dataset

```bash
wget https://ont-exd-int-s3-euwst1-epi2me-labs.s3.amazonaws.com/wf-human-variation/hg002%2bmods.v1/wf-human-variation-demo.tar.gz
tar -xzvf wf-human-variation-demo.tar.gz
```

Run with the demo data (including the new haplotype plot step):

```bash
nextflow run paudbr/wf-human-variation-paula \
    --bam wf-human-variation-demo/demo.bam \
    --ref wf-human-variation-demo/demo.fasta \
    --bed wf-human-variation-demo/demo.bed \
    --sample_name DEMO \
    --snp \
    --sv \
    --mod \
    --phased \
    -profile standard
```

## Pipeline overview

The workflow is composed of 6 distinct subworkflows, each enabled by a command line option:

* [SNP calling](#3-small-variant-calling-with-clair3): `--snp`
* [SV calling](#4-structural-variant-sv-calling-with-sniffles2): `--sv`
* [Modified base calling + haplotype plots](#5-modified-base-calling-with-modkit--haplotype-methylation-plots): `--mod`
* [CNV calling (Spectre)](#6a-copy-number-variants-cnv-calling-with-spectre): `--cnv`
* [CNV calling (QDNAseq)](#6b-copy-number-variants-cnv-calling-with-qdnaseq): `--cnv --use_qdnaseq`
* [STR genotyping](#7-short-tandem-repeat-str-genotyping-with-straglr): `--str`

### 1. Input and data preparation

The workflow relies on two primary input files:

1. A reference genome in [FASTA format](https://www.ncbi.nlm.nih.gov/genbank/fastaformat/)
2. Sequencing data in the form of a single [BAM file](https://samtools.github.io/hts-specs/SAMv1.pdf) or a folder of BAM files (aligned or unaligned).

When analysing human data, we strongly recommend using [human\_g1k\_v37.fasta.gz](https://ont-open-data.s3.amazonaws.com/references/human/G1Kv37/human_g1k_v37.fasta.gz) or [GCA\_000001405.15\_GRCh38\_no\_alt\_analysis\_set.fna.gz](https://ont-open-data.s3.amazonaws.com/references/human/GRCh38/GCA_000001405.15_GRCh38_no_alt_analysis_set.fna.gz). The workflow will automatically perform alignment if the input BAM contains unaligned reads (uBAM), or if the reference genome does not match the BAM header.

### 2. Data QC and pre-processing

The workflow starts by performing multiple checks of the input BAM file and computing:

1. Sequencing depth with [mosdepth](https://github.com/brentp/mosdepth)
2. Read alignment statistics with [fastcat](https://github.com/epi2me-labs/fastcat)

The workflow will check that the input BAM has a depth greater than `--bam_min_coverage`. Set `--bam_min_coverage 0` to skip this check. Use `--downsample_coverage true` and `--downsample_coverage_target <X>` (default: 60x) to downsample coverage before analysis.

### 3. Small variant calling with Clair3

Implements a deconstructed version of [Clair3](https://github.com/HKU-BAL/Clair3) (v1.0.4) to call germline SNPs and indels. The workflow automatically selects an appropriate Clair3 model from the input data headers. Variants are annotated with [SnpEff](https://pcingola.github.io/SnpEff/) and [ClinVar](https://www.ncbi.nlm.nih.gov/clinvar/). Phasing is available with `--phased`, using [whatshap](https://whatshap.readthedocs.io/).

### 4. Structural variant (SV) calling with Sniffles2

SV calling with [Sniffles2](https://github.com/fritzsedlazeck/Sniffles). A tandem repeat BED file can be provided with `--tr_bed` to improve calling in repetitive regions. SVs can be phased using `--phased`.

### 5. Modified base calling with modkit + haplotype methylation plots

Modified base calling is enabled with `--mod`. The workflow uses [modkit](https://github.com/nanoporetech/modkit) to aggregate 5mC counts from reads carrying `MM`/`ML` BAM tags (required). The default run uses `--cpg --combine-strands`. Use `--force_strand` for strand-aware output, or `--modkit_args` to fully customise the modkit invocation.

When `--phased` is also provided, modkit runs separately per haplotype, producing:
- Per-haplotype bedMethyl files: `{{ alias }}.wf_mods.1.bedmethyl.gz`, `{{ alias }}.wf_mods.2.bedmethyl.gz`
- Per-haplotype bigWig files: `{{ alias }}.wf_mods.1-5mC.bw`, `{{ alias }}.wf_mods.2-5mC.bw`

#### 🆕 Haplotype methylation comparison with deeptools

**Added in this fork.** After the per-haplotype bigWig files are generated, the workflow runs a custom deeptools step:

1. **`bigwigCompare`** computes the log2 ratio or arithmetic difference of 5mC signal between H1 and H2, producing a genome-wide haplotype difference bigWig (`{{ alias }}.wf_mods.haplo_diff.bw`). This file can be loaded directly in IGV to visualise allele-specific methylation differences.

2. **`computeMatrix`** + **`plotHeatmap`** / **`plotProfile`** generate summary visualisations of the haplotype methylation difference signal over regions of interest. If a `--bed` file is provided to the workflow, those regions are used; otherwise the plots are generated over gene bodies (using a built-in reference annotation).

This step only runs when both `--mod` and `--phased` are enabled.

### 6a. Copy number variants (CNV) calling with Spectre

CNV calling with an ONT implementation of [Spectre](https://github.com/epi2me-labs/ont-spectre/) (`--cnv`). Compatible with hg19/GRCh37 and hg38/GRCh38. Output: an annotated VCF. Use `--spectre_args` to override defaults.

### 6b. Copy number variants (CNV) calling with QDNAseq

Alternative CNV calling with [QDNAseq](https://github.com/ccagc/QDNAseq) (`--cnv --use_qdnaseq`). Recommended for shallow WGS or adaptive sampling data. Use `--qdnaseq_bin_size` to set bin size (default: 500 kbp).

### 7. Short tandem repeat (STR) genotyping with Straglr

STR genotyping with a fork of [Straglr](https://github.com/philres/straglr). Compatible with hg38/GRCh38 only. Provide `--sex XX` or `--sex XY` for correct chrX call counts. This subworkflow automatically enables `--snp` to generate the required haplotagged BAM.

### 8. Phasing variants

Enable with `--phased`. Uses [whatshap](https://whatshap.readthedocs.io/) to phase variants. The table below summarises which phased outputs are produced depending on the subworkflows selected:

| `--snp` | `--sv` | `--mod` | `--phased` | Phased SNP VCF | Phased SV VCF | Phased bedMethyl | Haplotype plots |
|---------|--------|---------|------------|:--------------:|:-------------:|:----------------:|:---------------:|
| ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| ✓ | ✓ |  | ✓ | ✓ | ✓ |  |  |
| ✓ |  |  | ✓ | ✓ |  |  |  |
|  | ✓ |  | ✓ |  | ✓ |  |  |
|  |  | ✓ | ✓ |  |  | ✓ | ✓ |

### 9. Variant annotation

Annotation is performed automatically by the SNP and SV subworkflows using [SnpEff](https://pcingola.github.io/SnpEff/) and [ClinVar](https://www.ncbi.nlm.nih.gov/clinvar/). Disable with `--annotation false` (required for non-human genomes).

### 10. Genome compatibility

| Genome | `--snp` | `--sv` | `--mod` | `--cnv` | `--cnv --use_qdnaseq` | `--str` | `--annotation false` | `--include_all_ctgs` |
|--------|---------|--------|---------|---------|----------------------|---------|----------------------|----------------------|
| hg19/GRCh37 | ✓ | ✓ | ✓ | ✓ | ✓ | | \* | |
| hg38/GRCh38 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | \* | |
| Other human | ✓ | ✓ | ✓ | | | | ✓ | |
| Non-human | ✓ | ✓ | ✓ | | | | ✓ | ✓ |

\* Annotation is on by default for hg19 and hg38 but can be disabled.

## Troubleshooting

* `--snp` and `--sv` annotations use [SnpEff](https://pcingola.github.io/SnpEff/); `--snp` also adds [ClinVar](https://www.ncbi.nlm.nih.gov/clinvar/) annotations. Variants classified as 'Benign' or 'Likely benign' will not appear in the report but are present in the output VCF.
* `--mod` requires reads basecalled with a modification-aware model providing `MM` and `ML` BAM tags.
* The haplotype deeptools plots (`haplo_diff.bw`, `haplo_heatmap.png`, `haplo_profile.png`) are only generated when **both** `--mod` and `--phased` are used.
* CRAM files generated by the workflow cannot be read without the corresponding reference.
* By default, SNPs and SVs are called on chromosomes 1–22, X, Y and MT. Enable `--include_all_ctgs` to call on all contigs.
* For non-human genomes: `--cnv false --str false --annotation false --include_all_ctgs true`.
* Ensure that the reference and any BED files use the same chromosome naming convention (with or without the `chr` prefix).
* The workflow expects an uncompressed or `bgzip`-compressed reference.
